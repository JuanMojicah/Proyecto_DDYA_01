Proyecto 1: Waze Universitario (Sistema de Navegación y Rutas)
Escenario:
Moverse por el campus universitario (o el sistema de transporte de la ciudad) en el primer día de clases es un caos. Su misión es construir un sistema de navegación por consola que permita a un usuario ingresar en qué edificio está y a qué edificio quiere ir, para devolverle la ruta exacta más corta y el tiempo estimado de llegada.

Requisitos:
•	Funcionamiento completo en consola: Todas las operaciones a ejecutar dentro del proyecto deben de ser realizadas por medio de consola, no es necesario generar una interfaz gráfica para el proyecto.

•	Operaciones básicas: El código debe soportar las siguientes operaciones básicas (esta es una guía y representación, el nombre de las funciones o parámetros dentro del código pueden variar):
    * Calcular ruta (punto de inicio, punto final): Mostrando el paso a paso de las indicaciones a seguir para poder llegar desde el punto inicial al final junto al tiempo invertido.
    * Agregar camino (punto de inicio, punto final, distancia): Agregar dentro de la estructura la posibilidad de incluir nuevos caminos entre puntos existentes.
    * Bloquear punto (punto): Por múltiples razones hay momentos en los que pasar por ciertos puntos de la universidad es imposible, poco conveniente o directamente inhabilitado. (Debe existir la contraparte de habilitar punto)
    * Calcular tiempo (punto de inicio, punto final): Dados el punto de inicio y de final, poder conocer el tiempo que tomaría el trayecto completo.

•	Manejo del proyecto: El proyecto debe de existir dentro de un repositorio de GitHub en donde ambos integrantes deben de realizar commits significativos durante el tiempo de trabajo del proyecto.
•	Consideraciones de la información: Los usuarios no buscan el nodo "4", buscan la "Biblioteca". La implementación completa de la estructura debe de ser capaz de poder identificar cada uno de estos casos de la forma más conveniente.
•	Casos de justificación: Deben de contar con un archivo de input de no menos de 100 líneas en las que el código realizado reciba órdenes y sea completamente capaz de ejecutarlas CORRECTAMENTE.
•	Resultados: La nota resultante del proyecto será impartida y definida única y exclusivamente durante el momento de la sustentación, las respuestas, justificación del código y ejecución del mismo serán los factores clave de la misma.
